## 一、追加背景

主项目的截图工具已完成 Stage 1-5 并验证可用。现追加"贴图"功能（Snipaste 风格），作为 **Stage 6** 进行开发。

### 功能定位
将截图作为无边框置顶图片"钉"在屏幕上，跨窗口持续可见，用作视觉参考。常见场景：
- 截一段代码贴出来，切到另一个文件对照着写
- 贴参考图在屏幕角落，画图 / 写文档时对照
- 开会时贴 PPT 某页，边看边记笔记

## 二、与现有功能的关系

### 2.1 不引入新的全局快捷键
**关键设计原则**：贴图功能不新增任何全局快捷键，而是**复用现有截图快捷键，用"按键时长"区分两种行为**。

| 行为 | 触发方式 | 效果 |
|---|---|---|
| 普通截图 | 短按截图快捷键（< 500ms 内松开） | 截图并存盘（原 F1 / F2 行为） |
| 截图 + 贴图 | 长按截图快捷键（> 500ms） | 截图并存盘 + 在原位置贴出图片 |

这意味着原有的所有截图快捷键（每个预设的快捷键 + 手动框选快捷键）都获得"长按贴图"能力，**不增加用户的快捷键记忆负担**。

### 2.2 不影响存盘流程
贴图触发时，截图本体仍按 F6 规则正常保存到截图文件夹。**贴图是"额外动作"，不替代存盘**。

### 2.3 贴图操作不抢全局焦点
贴图本身的所有操作（缩放、移动、关闭等）**只在鼠标悬停在贴图上时响应**，不是全局快捷键。因此不会与用户在其他软件中的 `Ctrl+S` 等冲突。

## 三、功能需求

### F15. 截图并贴图（长按触发）

- 长按任意截图快捷键超过 **500ms**（可在高级设置中调整），触发"截图 + 贴图"
- 短按（< 500ms 内松开）仍是普通截图，不贴图
- 长按触发时：
  1. 执行原有截图逻辑（与该快捷键绑定的预设行为一致，固定区域或框选）
  2. 截图正常存盘
  3. 同时将该截图作为无边框置顶窗口贴在**原截图位置**
- 可同时存在多张贴图，互不干扰

### F16. 贴图操作（鼠标悬停时响应）

每张贴图独立响应以下操作。**这些操作不是全局快捷键，仅在鼠标位于贴图上时生效**：

| 操作 | 效果 | 范围 / 备注 |
|---|---|---|
| 左键按下并拖动（移动 > 阈值像素） | 移动该贴图 | 默认阈值 5px |
| 左键单击（不拖动） | 选中该贴图，并置于最顶层 | |
| 左键双击 | 关闭该贴图 | |
| 滚轮 | 缩放 | 范围 20%-500% |
| Ctrl + 滚轮 | 调整透明度 | 范围 10%-100% |
| 右键单击 | 弹出操作菜单 | 见 F19 |

### F17. 贴图选中状态

- 默认**无视觉反馈**（保持图片纯净）
- 同一时间只有一张贴图处于选中状态
- 选中任意贴图会自动将其层级提升到最顶层
- 视觉反馈样式可在高级设置中切换（关闭 / 细描边 / 阴影发光）

### F18. 贴图焦点行为

- 鼠标悬停在贴图上即可响应滚轮等事件，**无需先点击获取焦点**
- 贴图窗口**不抢键盘焦点**（使用 `WS_EX_NOACTIVATE` 实现）
- 用户在其他窗口工作时，鼠标移到贴图上滚轮即生效，原窗口焦点不丢失

### F19. 右键菜单（操作兜底）

右键单击任意贴图弹出菜单，作为所有操作的"可见入口"。菜单项展示当前操作方式作为视觉提示：

```
┌────────────────────────────┐
│ 复制到剪贴板               │
│ ────────────────────────── │
│ 关闭                双击   │
│ 关闭所有贴图               │
│ ────────────────────────── │
│ 透明度          Ctrl+滚轮  │
│ 缩放                滚轮   │
│ 移动          左键拖动     │
│ ────────────────────────── │
│ 重置大小（100%）           │
│ 重置透明度（100%）         │
└────────────────────────────┘
```

设计意图：
- **快捷键给老手用**，菜单给所有人兜底
- 菜单本身**显示快捷键提示**，等于教学
- "关闭所有贴图"**不设快捷键**，只能通过菜单触发，避免误删

### F20. 贴图层级管理

- 新贴出的图默认在所有贴图的最顶层
- 用户点击某张贴图时，该贴图浮到所有贴图的最顶层
- 所有贴图都在普通窗口之上（包括悬浮球，但悬浮球操作不受贴图遮挡——见 F22）

### F21. 截图时的贴图处理

- **默认行为**：触发截图前自动隐藏所有贴图，截图完成后恢复显示
- 此行为可在高级设置中关闭，让贴图也能被新截图捕获

### F22. 贴图与悬浮球的关系

- 贴图层级在悬浮球之上（视觉上贴图可能盖住悬浮球）
- 悬浮球的右键菜单新增以下选项：
  - "隐藏所有贴图" / "显示所有贴图"（切换显示状态，不删除）
  - "关闭所有贴图"（确认后删除全部）
  - "当前贴图数：N"（状态显示）

### F23. 贴图位置与多显示器

- 第一版仅保证主显示器内贴图行为正确
- 用户拖动贴图超出屏幕边缘时，**允许超出**（不强制吸回），但下次启动不恢复（贴图不持久化）
- 关闭工具时所有贴图直接销毁，**不保存贴图状态**

## 四、高级设置项（追加）

以下选项归入设置面板的"高级"区域：

| 设置项 | 默认值 | 取值范围 |
|---|---|---|
| 长按触发贴图的时长 | 500ms | 300-1000ms |
| 双击判定时间窗 | 500ms | 200-800ms |
| 选中视觉反馈 | 关闭（无反馈） | 关闭 / 细描边 / 阴影发光 |
| 拖动判定阈值 | 5px | 1-20px |
| 截图时隐藏贴图 | 开启 | 开启 / 关闭 |
| 最低透明度 | 10% | 0-50% |
| 最低缩放比例 | 20% | 5-50% |
| 最高缩放比例 | 500% | 200-1000% |

## 五、技术实现要点

### 5.1 长按检测

- 在快捷键监听层增加按键时长追踪
- 按下时记录时间戳，松开时计算差值
- 差值 < 阈值 → 短按 → 普通截图
- 差值 ≥ 阈值 → 长按 → 截图 + 贴图
- **注意**：长按检测要在按键 *松开* 时触发动作，不能在按下时就触发（否则无法区分短按长按）
- 用户体验上要有提示：长按到达阈值时可以播放轻微的视觉 / 听觉反馈（可选，提示用户"已进入贴图模式"）

### 5.2 贴图窗口实现

- 每张贴图 = 一个独立的 tkinter `Toplevel` 窗口
- 设置：`overrideredirect(True)`（去边框）+ `-topmost True`（置顶）+ `-alpha`（透明度）
- 使用 `pywin32` 设置 `WS_EX_NOACTIVATE` 避免抢焦点
- 缩放：通过 `Pillow` resize 后重新载入图片，或对 Label 缩放

### 5.3 鼠标悬停事件响应（不抢焦点）

- 关键挑战：窗口设置 `WS_EX_NOACTIVATE` 后，常规的键盘事件可能不响应
- 滚轮事件可以通过 `<MouseWheel>` 在窗口未激活时捕获（Windows 上 OK）
- 但 `Ctrl+滚轮` 中的 Ctrl 状态需要用 `event.state` 检测，而非依赖键盘焦点
- 全局鼠标位置可用 `pyautogui.position()` 或 `pywin32` 的 `GetCursorPos` 查询

### 5.4 多贴图管理

- 用一个全局列表 / 字典管理所有贴图实例
- 提供方法：`hide_all()`、`show_all()`、`close_all()`、`bring_to_front(pin_id)`
- 截图前调用 `hide_all()`，截图完成后 `show_all()`

### 5.5 内存与性能

- 每张贴图持有一份原始图片数据 + 当前显示的缩放后图片
- 大尺寸贴图（如 4K 截图）多张贴出时要注意内存占用
- 建议限制最大贴图数量（如 20 张），超过时提示或自动关闭最旧的

## 六、明确不做的事

为防止本功能范围蔓延，**第一版贴图不做**：

1. ❌ 贴图旋转
2. ❌ 在贴图上画箭头 / 写字 / 马赛克（标注功能）
3. ❌ 贴图鼠标穿透模式（点击穿过到下方窗口）
4. ❌ 贴图状态持久化（重启后恢复）
5. ❌ 贴图分组管理
6. ❌ 从剪贴板贴图（按 `Ctrl+V` 把剪贴板里的图贴出来）—— 这个其实很有用，但放到下一个迭代
7. ❌ 贴图边缘吸附 / 对齐辅助线
8. ❌ 贴图缩略图栏 / 管理面板

## 七、已知风险与对策（追加）

| 风险 | 等级 | 说明 | 对策 |
|---|---|---|---|
| 长按延迟影响普通截图体验 | 🟡 中 | 用户短按时也要等 500ms 才能确认是短按吗？ | 短按在松开瞬间立即触发普通截图，无延迟；长按是在按住 500ms 时即可触发贴图标志（松开后再贴出来）|
| 鼠标悬停事件不响应 | 🟡 中 | `WS_EX_NOACTIVATE` 可能影响事件捕获 | 原型阶段必须先验证滚轮和 Ctrl 状态能正确捕获 |
| 大量贴图导致内存膨胀 | 🟡 中 | 贴 10+ 张 4K 截图可能占用数百 MB | 设置最大数量限制；超过提示 |
| 贴图被新截图截到 | 🟢 低 | 已在 F21 处理 | 默认隐藏，可配置 |
| 多贴图层级混乱 | 🟢 低 | 多个贴图都 `topmost` 时层级由系统决定 | 点击哪张哪张 `lift()`，依赖 tkinter 层级管理 |
| 长按和短按的混淆 | 🟢 低 | 用户可能无意中按久了，触发了不想要的贴图 | 长按阈值可调；菜单里方便关闭 |
