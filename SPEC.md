## 一、项目概述

### 1.1 项目名称
（暂称"截图工具"，待定正式名）

### 1.2 项目定位
一款 Windows 平台的轻量级桌面截图工具，以"固定区域反复截图"为核心场景，辅以手动框选。
工具以**悬浮球**形态常驻桌面，通过快捷键快速触发截图，自动按时间戳命名并存入指定文件夹。

### 1.3 目标用户
- 主要用户：开发者本人（自用）
- 次要用户：开发者的朋友（少量分发，3-10 人量级）

### 1.4 设计哲学
- **少打扰**：常驻但不碍事，悬浮球可拖到屏幕任意位置
- **快触发**：快捷键一键完成"截图 + 命名 + 存盘"全流程
- **可配置**：路径、坐标、快捷键全部用户可改
- **能维护**：使用开发者熟悉的技术栈，便于长期演进

## 二、功能需求

### 2.1 核心功能

#### F1. 固定区域截图
- 用户预设一个矩形区域（左上角坐标 + 宽高）
- 按下绑定的快捷键，立即截取该区域并存盘
- 支持多个预设，每个预设独立的坐标和快捷键

#### F2. 手动框选截图
- 按下绑定的快捷键，进入框选模式
- 屏幕变为半透明遮罩，用户拖动鼠标画框
- 松开鼠标后立即截图并存盘
- 按 ESC 取消

#### F3. 屏幕标定（设置固定区域用）
- 在设置面板中点击"标定"按钮
- 主窗口隐藏，进入和 F2 类似的框选模式
- 用户拖出一个框
- 坐标自动回填到设置面板，无需手工查像素

#### F4. 自定义保存路径
- 用户可在设置中指定截图保存文件夹
- 支持点击按钮调用系统文件夹选择对话框
- 启动时若路径不存在，自动创建

#### F5. 自定义快捷键
- 每个预设的快捷键可独立配置
- 手动框选模式的快捷键也可配置
- 注册失败（冲突）时，提示用户更换

#### F6. 文件命名
- 默认格式：`screenshot_YYYYMMDD_HHMMSS.png`
- 同一秒内多次截图自动追加序号：`_001`、`_002`

### 2.2 形态需求（悬浮球）

#### F7. 悬浮球外观
- 圆形小窗口，无边框
- 视觉风格：简洁、克制、不土
- 显示信息：一个图标（或简单状态指示）
- 大小：默认 48-64 像素直径（可调）

#### F8. 悬浮球交互
- **拖动**：按住鼠标左键可拖动到屏幕任意位置
- **单击**：弹出 / 收起设置面板（面板在悬浮球旁边显示）
- **右键**：弹出菜单（设置、隐藏、退出、关于）
- **双击**（可选）：触发默认预设的截图

#### F9. 悬浮球位置记忆
- 关闭时记录位置
- 下次启动时恢复到上次位置

#### F10. 设置面板
- 单击悬浮球后展开
- 包含：预设列表、坐标编辑、快捷键编辑、保存路径、标定按钮、测试截图按钮
- 使用 customtkinter 实现，深色 / 浅色模式可选
- 不追求灵动岛形变动画（明确不做）

### 2.3 工程需求

#### F11. 配置文件
- 格式：JSON
- 位置：`%APPDATA%\<工具名>\config.json`
- 包含所有用户配置：预设列表、保存路径、快捷键、悬浮球位置、主题等

#### F12. 打包分发
- 使用 PyInstaller 打包为单文件 exe
- 双击即用，无需安装 Python
- 接受 30-50MB 的体积

#### F13. 单实例运行
- 防止重复启动多个进程导致快捷键互抢
- 已运行时再次启动应直接退出或激活已有窗口

#### F14. 开机自启（可选功能，后期）
- 用户在设置中开关
- 通过写入注册表实现

## 三、非功能需求

### 3.1 平台
- Windows 10 / Windows 11
- 暂不支持 macOS / Linux

### 3.2 显示器支持
- **第一版仅完整支持主显示器**
- 多显示器场景下行为不保证，后期迭代

### 3.3 DPI 缩放
- 必须正确处理 Windows 100% / 125% / 150% / 175% / 200% 缩放
- 启动时调用 `SetProcessDpiAwareness`，全程使用物理像素

### 3.4 性能
- 截图触发到完成存盘 < 200ms
- 悬浮球 CPU 占用 < 1%（空闲时）

### 3.5 可维护性
- 代码模块化：截图逻辑、配置管理、UI、快捷键监听分离
- 关键逻辑有注释
- 配置文件向后兼容（新版本能读旧版配置）

## 四、技术栈

| 类别 | 选型 | 用途 |
|---|---|---|
| 语言 | Python 3.10+ | 主语言 |
| 截图 | `mss` | 高速屏幕抓取 |
| 全局快捷键 | `keyboard` | 监听快捷键（注意可能需管理员权限） |

| 打包 | `PyInstaller` | 生成单文件 exe |
| 版本管理 | `git` | 必须从第一天使用 |

## 五、明确不做的事（项目边界）

为防止范围蔓延，以下功能**第一版不做**，避免精力分散：

1. ❌ macOS / Linux 支持
2. ❌ 多显示器完整适配
3. ❌ 灵动岛形变动画
4. ❌ 截图后的标注、马赛克、文字编辑
5. ❌ OCR 文字识别
6. ❌ 自动上传图床 / 云同步
7. ❌ 截图历史浏览器
8. ❌ 录屏 / GIF
9. ❌ 多语言（第一版只做中文）
10. ❌ 自动更新机制
11. ❌ 数字证书签名

这些功能未来可以加，但**不在 MVP 范围**。

## 六、实施计划（分阶段）

### Stage 1：核心截图链路（无 GUI）
**目标**：验证截图、快捷键、存盘的主链路通畅。

- 命令行启动
- 配置写死在代码顶部（坐标、路径、快捷键）
- 单一固定区域截图
- 手动框选截图（用 tkinter 全屏遮罩实现）
- 时间戳命名
- 存到指定文件夹

**完成标准**：能跑、按快捷键能截、文件能正确保存。
**预计时长**：1-2 个晚上。

### Stage 2：配置化
**目标**：把硬编码改成配置文件，支持多预设。

- 引入 `config.json`
- 多预设支持（数组结构）
- 每个预设独立快捷键
- 配置文件放 `%APPDATA%`
- 启动时读取，缺失则生成默认配置

**完成标准**：手动编辑 config.json 就能改所有配置，重启生效。

### Stage 3：传统 GUI（customtkinter）
**目标**：做一个能编辑配置的设置窗口。

- 预设列表的增删改
- 坐标输入框
- 快捷键设置控件
- 保存路径选择按钮
- "标定"按钮（屏幕拖框 → 自动填坐标）
- "测试截图"按钮
- DPI 处理
- **此阶段产物已可发朋友试用**

**完成标准**：朋友拿到能配出自己的预设并使用。

### Stage 4：悬浮球形态
**目标**：把传统窗口改造成悬浮球 + 弹出面板。

- 圆形无边框窗口
- 拖动、单击展开、右键菜单
- 位置记忆
- 设置面板在悬浮球旁弹出
- 焦点不抢夺（`WS_EX_NOACTIVATE`）

**完成标准**：悬浮球能拖、能展开、能用。

### Stage 5：分发与工程优化
**目标**：让工具能稳定地发给朋友用。

- PyInstaller 打包（带 / 不带 console 两个版本）
- 单实例运行
- 配置文件迁移逻辑（新版能读旧版）
- 杀毒白名单说明文档
- 简单使用说明（README）
- 开机自启选项

**完成标准**：朋友收到 exe，双击能用，能配置，能截图。

## 七、已知风险与对策

| 风险 | 等级 | 说明 | 对策 |
|---|---|---|---|
| `keyboard` 库需管理员权限 | 🔴 高 | 某些场景下普通权限快捷键失灵 | 打包时考虑 `--uac-admin`；或备选 `pynput` |
| 杀毒软件误报 | 🔴 高 | 键盘监听 + 屏幕抓取被视为可疑 | 开源 + 教朋友加白名单；不买证书 |
| 悬浮球抢焦点 | 🟡 中 | 无边框置顶窗口默认抢键盘焦点 | 用 `pywin32` 设 `WS_EX_NOACTIVATE` |
| 多显示器坐标 | 🟡 中 | 副屏可能负坐标 | 第一版只保证主屏 |
| DPI 缩放错位 | 🟡 中 | 截图坐标和实际像素不符 | 启动设 DPI awareness |
| 全屏程序冲突 | 🟡 中 | 看视频 / 游戏时悬浮球碍事 | 检测前台窗口全屏则隐藏；或提供临时隐藏快捷键 |
| 截图把悬浮球截进去 | 🟢 低 | 固定区域覆盖到悬浮球位置 | 截图前瞬间隐藏 |
| 快捷键冲突 | 🟢 低 | 用户设了已被占用的键 | 注册失败时提示更换 |
| 配置文件位置 | 🟢 低 | 放 exe 同目录可能没写权限 | 放 `%APPDATA%` |
| PyInstaller 启动黑窗 | 🟢 低 | 打包后控制台一闪 | 发布版用 `--noconsole` |

## 八、开发流程约定

### 8.1 Vibe Coding 原则
- **每次开新对话先丢本文档作为上下文**
- AI 写代码后，开发者必须实际运行验证
- 每个 Stage 完成后 commit，并更新 `NOTES.md`
- 单一对话内不连续推进超过一个 Stage
- 当 AI 在某问题上反复改不对超过 3-4 轮，**重启对话**，带上代码和问题重新描述

### 8.2 Git 约定
- 第一天就 init 仓库
- 每个 Stage 完成打 tag（`v0.1`、`v0.2` ...）
- 重大重构前建分支

### 8.3 配套文档
- `SPEC.md`（本文档）：项目宪法，长期稳定
- `NOTES.md`：踩坑记录、临时决策、未解之谜
- `CHANGELOG.md`：版本变更记录
- `README.md`：给朋友看的使用说明（Stage 5 才写）

### 8.4 代码组织（建议结构）
```
screenshot-tool/
├── SPEC.md
├── NOTES.md
├── README.md
├── requirements.txt
├── src/
│   ├── main.py              # 入口
│   ├── config.py            # 配置加载 / 保存
│   ├── capture.py           # 截图逻辑
│   ├── hotkey.py            # 快捷键监听
│   ├── ui/
│   │   ├── floating_ball.py # 悬浮球
│   │   ├── settings.py      # 设置面板
│   │   └── selector.py      # 框选 / 标定遮罩
│   └── utils.py
└── assets/
    └── icon.png
```
